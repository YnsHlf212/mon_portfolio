Vue d'ensemble

RoomBooking est une application web de réservation de salles développée avec Symfony 7.3. Elle permet aux étudiants et coordinateurs de réserver des salles équipées, avec un système de détection de conflits de créneaux, une gestion fine des rôles (étudiant, coordinateur, admin) et un dashboard statistique pour les administrateurs.

Modèle de données

L'application repose sur 5 entités Doctrine avec des relations complexes :

User
  • email, firstName, lastName
  • password (hashé bcrypt)
  • roles (JSON)
  • promotion (FK)
Reservation
  • room (FK), owner (FK User)
  • bookedFor (FK User)
  • startDatetime / endDatetime
  • createdAt / cancelledAt
Room
  • name, capacity, description
  • isActive
  • equipments (M2M)
  • reservations (1TM)
Equipment / Promotion
  • Equipment : name, description, rooms (M2M)
  • Promotion : name, users (1TM)

Fonctionnalités principales

Authentification et Sécurité
  • Authentification form_login Symfony avec protection CSRF stateless
  • Hashage bcrypt automatique des mots de passe
  • Hiérarchie de rôles avec contrôle d'accès granulaire
Réservation intelligente
  • Détection de conflits de réservation par requête DQL personnalisée (chevauchement de créneaux)
  • Validation côté serveur (Symfony Validator) et côté client (JS vanilla)
  • Interdiction de réserver dans le passé, date de fin après date de début
  • Envoi d'email de confirmation via Symfony Mailer avec template HTML
Dashboard et Interface
  • Dashboard personnalisé par rôle : stats admin, réservations à venir et historique
  • Recherche temps réel sur les tableaux (filtre JS client-side)
  • Interface responsive avec menu hamburger mobile
  • Flash messages avec disparition automatique (4s)

Aspects Techniques

Détection de conflits de créneaux

Requête DQL personnalisée avec logique de chevauchement (start < end ET end > start), exclusion de la réservation en cours d'édition pour éviter les faux conflits.

Relations complexes Doctrine

Relations Many-to-Many (Room ↔ Equipment), One-to-Many (Promotion → Users, Room → Reservations), gestion de la suppression avec garde-fous (promotion avec étudiants, salle avec réservations actives).

Migrations et DataFixtures

Versionnage du schéma BDD via Doctrine Migrations. Jeu de données de développement complet via DataFixtures (admin, coordinateur, étudiant, salles, équipements, promotion, réservation test).

Stack technique détaillée

Front-end

Apprentissages Clés

Framework Symfony

Maîtrise de l'écosystème Symfony : bundles, services, sécurité, formulaires, validation, envoi d'emails et gestion multi-environnements (.env).

ORM Doctrine

Conception d'entités avec relations complexes (M2M, 1TM), requêtes DQL personnalisées, migrations automatiques et fixtures de développement.

Sécurité applicative
